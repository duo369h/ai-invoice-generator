# Safe Copy Rewrite

Date: 2026-07-01
Scope: Copy recommendations only. No code changes.

## Rewrite Principles

- Keep primary identity: freelancer workflow, client management, proposals, quotes, document delivery.
- Move invoice/payment language below workflow/product language.
- Keep Paddle references only as checkout/legal disclosure.
- Avoid "payment processor", "money movement", "get paid", "revenue", "balance", "transfer", and "financial OS" framing.

## Recommended Page-Level Direction

### Landing

Primary narrative should become:

```text
Corvioz helps freelancers organize quotes, proposals, client documents, and client records in one workspace.
```

Avoid leading with:

```text
get paid
paid work
payment links
revenue outcomes
billing flow
```

### Pricing

Pricing can mention subscription checkout and refund terms, but the cards should describe plan scope as:

```text
document workflow
client records
client portal
clean export
team/studio operations
```

Avoid making plan value depend on:

```text
payment success
payment follow-up
invoice ROI
revenue outcomes
```

### Footer / Legal Trust

Use:

```text
Paid plan checkout is handled securely by Paddle.
```

Avoid:

```text
Secure subscription billing via Paddle
get paid faster
payment processing
billing infrastructure
```
