# Corvioz Domain DNS Checklist

Scope: production domain readiness for the first Vercel deployment.

Canonical production URL:

```text
https://corvioz.com
```

Secondary URL:

```text
https://www.corvioz.com
```

## Domain Targets

| Domain | Role | Expected behavior |
| --- | --- | --- |
| `corvioz.com` | Apex canonical domain | Loads the Vercel production deployment over HTTPS. |
| `www.corvioz.com` | Secondary domain | Redirects permanently to `https://corvioz.com`. |

## Vercel Domain Setup

In Vercel Project Settings -> Domains:

1. Add `corvioz.com`.
2. Add `www.corvioz.com`.
3. Set `corvioz.com` as the primary domain.
4. Configure `www.corvioz.com` to redirect to `corvioz.com`.
5. Wait until both domains show valid configuration in Vercel.

## DNS Records

Use the exact DNS records shown by Vercel for the project.

If using Vercel-managed nameservers:

- Point the registrar nameservers to Vercel.
- Let Vercel manage apex and `www` records.

If using external DNS:

- Configure the apex `corvioz.com` record exactly as Vercel instructs.
- Configure `www.corvioz.com` exactly as Vercel instructs.
- Do not guess DNS target values from another project.

## SSL Checklist

curl -I https://www.corvioz.com

## Canonical Checklist

| Check | Expected result |
| --- | --- |
| `NEXT_PUBLIC_SITE_URL` | Set to `https://corvioz.com` in Vercel Production. |
| Sitemap | `https://corvioz.com/sitemap.xml` contains `https://corvioz.com` URLs. |
| Robots | `https://corvioz.com/robots.txt` references `https://corvioz.com/sitemap.xml`. |
| Metadata | Canonical metadata uses apex domain, not `www`. |
| Public app links | Portal/card links use the apex canonical base when generated by production code. |

Suggested checks:

```bash
curl https://corvioz.com/sitemap.xml
curl https://corvioz.com/robots.txt
```

## WWW Redirect Checklist

| URL | Expected result |
| --- | --- |
| `https://www.corvioz.com/` | Redirects to `https://corvioz.com/`. |
| `https://www.corvioz.com/invoice-generator` | Redirects to `https://corvioz.com/invoice-generator`. |
| `https://www.corvioz.com/quote-generator` | Redirects to `https://corvioz.com/quote-generator`. |

Suggested commands:

```bash
curl -I https://www.corvioz.com/
curl -I https://www.corvioz.com/invoice-generator
curl -I https://www.corvioz.com/quote-generator
```

Pass criteria:

- Redirect status is permanent or Vercel-managed canonical redirect behavior.
- Redirect target keeps the path.
- Redirect target uses `https://corvioz.com`, not `http` and not `www`.
